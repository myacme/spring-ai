## Java 代码示例

### 1. 基础聊天

```java
@Service
@RequiredArgsConstructor
public class ChatService {
    
    private final ChatClient chatClient;
    
    public String chat(String message) {
        return chatClient.prompt()
                .user(message)
                .call()
                .content();
    }
}
```

### 2. 流式聊天

```java
public Flux<String> chatStream(String message) {
    return chatClient.prompt()
            .user(message)
            .stream()
            .content();
}
```

### 3. 文本分词

```java
@Service
public class TextSplitterService {
    
    // TokenTextSplitter
    public List<Document> splitByToken(String text) {
        TokenTextSplitter splitter = new TokenTextSplitter(500, 200, 5, 1000, false);
        Document document = new Document(text);
        return splitter.apply(Arrays.asList(document));
    }
    
    // SentenceSplitter (更适合中文)
    public List<Document> splitBySentence(String text) {
        SentenceSplitter splitter = new SentenceSplitter(300);
        Document document = new Document(text);
        return splitter.split(Arrays.asList(document));
    }
}
```

### 4. Function Calling

```java
@Service
public class WeatherToolService {
    
    private final ChatClient chatClient;
    
    @Tool(description = "获取指定城市的天气信息")
    public String getWeather(String city) {
        // 实现天气查询逻辑
        return "天气信息";
    }
    
    public String queryWeather(String userMessage) {
        return chatClient.prompt()
```

## 配置说明

### application.yml

```yaml
server:
  port: 8083

spring:
  ai:
    dashscope:
      api-key: ${DEEPSEEK_API_KEY:your_api_key}
      chat:
        options:
          model: deepseek-chat
          temperature: 0.7      # 温度参数 (0-1)
          max-tokens: 2048      # 最大 token 数
          top-p: 0.9            # 核采样参数
```

### 关键参数说明

- **temperature**: 控制输出的随机性
  - 0.0: 更确定性，适合事实性问题
  - 0.7: 平衡创造性和准确性
  - 1.0: 更具创造性，适合创意写作

- **max-tokens**: 限制响应长度
  - 短回答: 512-1024
  - 中等回答: 1024-2048
  - 长回答: 2048-4096

- **top-p**: 控制词汇选择的多样性
  - 0.9: 推荐值，平衡多样性和准确性

## 最佳实践

### 1. 文本分词参数选择

**TokenTextSplitter:**
- 块大小: 300-700 tokens
- 最小字符数: 200-400
- 中文场景: keepSeparator=false

**SentenceSplitter:**
- 块大小: 200-500 tokens
- 更适合中文，能保持语义完整性

## 常见问题

### Q1: 如何选择合适的分词器？
- **英文文本**: TokenTextSplitter
- **中文文本**: SentenceSplitter（推荐）
- **混合文本**: 根据主要语言选择

### Q2: Function Calling 不生效怎么办？
1. 确保方法有 `@Tool` 注解
2. 检查 description 是否清晰
3. 调用时通过 `.functions()` 注册
4. 查看日志确认函数是否被调用

### Q3: 如何提高分词质量？
1. 调整块大小参数
2. 使用 SentenceSplitter 处理中文
3. 添加重叠区域避免语义断裂
4. 在自然边界（句号、段落）处分割

### Q4: 流式响应如何使用？
```java
chatClient.prompt()
    .user(message)
    .stream()
    .content()
    .subscribe(chunk -> {
        System.out.print(chunk);  // 逐块输出
    });
```

## 性能优化建议

1. **缓存常用响应**: 对于重复问题使用缓存
2. **批量处理**: 多个文本分词可以批量进行
3. **异步处理**: 使用 WebFlux 提高并发能力
4. **合理设置超时**: 避免长时间等待
